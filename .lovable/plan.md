## Что меняем

Три улучшения для тестов в `TestPage.tsx` и `YakuniyTestPage.tsx`:

### 1. Авто-переход через 1.5 секунды после выбора ответа

После выбора ответа (зелёный/красный фидбек уже виден):
- Через **1500мс** автоматически переход к следующему вопросу
- На последнем вопросе — авто-показ экрана результата
- Кнопка **"Кейинги" удаляется** (больше не нужна — переход автоматический)
- Кнопка **"Олдинги" остаётся** на своём месте — пользователь может вернуться и пересмотреть предыдущий вопрос

При возврате через "Олдинги" к уже отвеченному вопросу — таймер авто-перехода **не запускается повторно** (ответ уже был выбран ранее, повторный select не происходит). Пользователь спокойно смотрит и нажимает "Олдинги" дальше или ждёт... — но таймера нет, поэтому застрянет. Решение: при возврате назад показываем кнопку **"Кейинги"** только если на текущем вопросе уже есть ответ И мы вернулись назад (т.е. это не свежий выбор). Проще: **показывать "Кейинги" всегда когда `hasAnswered === true`, но таймер запускать только при свежем выборе ответа**. Так:
- Свежий ответ → виден фидбек → 1.5с → авто-переход (кнопка "Кейинги" видна, но не нужна)
- Возврат назад через "Олдинги" → ответ уже стоит → "Кейинги" работает как раньше, таймера нет

В режиме "работа над ошибками" таймер на правильный ответ **поднимаем 900мс → 1500мс** для единообразия.

### 2. Зум картинки по клику

В `QuestionView.tsx`:
- Клик по картинке открывает **полноэкранный модал** (shadcn Dialog) с увеличенной картинкой
- Размер: `max-w-[95vw] max-h-[95vh]`, `object-contain`
- Крестик (×) для закрытия в углу (встроен в `DialogContent`), Esc и клик по фону тоже закрывают
- Курсор `cursor-zoom-in` на картинке-подсказке
- Зум активен **только если у вопроса есть реальная картинка** (`imageUrl`); логотип-заглушку не зумим

### 3. Кнопка "Работа над ошибками" — всегда при наличии ошибок

Меняем условие на экране результата (в обоих файлах):

**Было:** кнопка появляется только при `score >= 95 && score < 100`.

**Станет:** кнопка появляется **всегда когда `wrongQuestionIds.length > 0`**.

Расклад на экране результата:

- **100%** → "Мавзуларга қайтиш"
- **95–99%** → "Xatolar ustida ishlash" (primary) + "Мавзуларга қайтиш"
- **<95% c ошибками** → "Xatolar ustida ishlash" (primary) + "Қайта бошлаш" (outline) + "Мавзуларга қайтиш" (outline)
  - Новый сценарий: например 60% — выбор пройти заново всё или только те 40% что ошибся

Прогресс темы при работе над ошибками **не меняем** (остаётся как был). Чтобы разблокировать следующую тему, нужно 95%+ в обычном прохождении.

## Что НЕ трогаем
- Подсчёт score и сохранение прогресса
- Edge functions, БД
- Правила разблокировки (95%+)
- Кнопку "Олдинги" — остаётся
- Видео, авторизация

## Технические детали

**`QuestionView.tsx`:**
- Локальный стейт `zoomOpen`
- Картинка оборачивается в `<button>` (только когда `imageUrl` есть) с `cursor-zoom-in`
- Рядом `<Dialog open={zoomOpen} onOpenChange={setZoomOpen}>` с `DialogContent className="max-w-[95vw] max-h-[95vh] p-0 bg-black/95 border-none"`, внутри `<img>` с `object-contain w-full h-full`

**`TestPage.tsx` / `YakuniyTestPage.tsx`:**
- `useRef<NodeJS.Timeout | null>(null)` для таймера
- В `handleSelectAnswer`: после `setAnswers`, **только если ответа на этот вопрос ещё не было**, запускаем `timerRef.current = setTimeout(handleNext, 1500)`
- Cleanup: `clearTimeout` при размонтировании и при смене `currentIndex` (`useEffect` зависимый от `currentIndex`)
- Из JSX удаляем кнопку "Кейинги" (правую). Левая "Олдинги" — остаётся; контейнер делаем `flex justify-start`
- В блоке `isFinished`: `const canReviewMistakes = wrongQuestionIds.length > 0`
- В режиме mistake таймер 900 → 1500мс

**Граничные случаи:**
- Нажатие "Орқага" во время отсчёта — таймер очищается в cleanup
- Возврат через "Олдинги" к отвеченному вопросу — таймер не перезапускается (проверка "ответ уже был")
- Последний вопрос: авто-`handleNext` корректно завершает тест и показывает экран результата
